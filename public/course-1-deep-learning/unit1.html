Lesson 1

The perceptron and the linear decision boundary

In the summer of 1958, a Cornell psychologist named Frank Rosenblatt stood a room-sized machine called the Mark I Perceptron in front of the press and described it as the embryo of a device that would one day walk, talk, and be conscious of its own existence. Strip away the theater and the Mark I was a 20-by-20 grid of cadmium-sulfide photocells wired through 512 motor-driven potentiometers — physical dials, each one a connection strength that could be turned up or down. Show it an image, and the photocells drove voltages forward through those adjustable dials to a bank of output units that fired or didn't. Turn the dials, and the same photocells produced a different verdict. That machine was solving, in hardware, exactly the problem this lesson solves in a dozen lines of NumPy: given a set of measurements, find a straight cut through the space that separates one category from another.

Rosenblatt's real contribution wasn't the wiring, it was the learning rule: the dials adjusted themselves from labeled examples, with no engineer hand-tuning them. That felt like magic in 1958. It did not survive the decade. In 1969, Marvin Minsky and Seymour Papert published Perceptrons, a precise mathematical account of what a single perceptron can and cannot separate. Their signature result was almost insultingly simple: a perceptron cannot learn the XOR function — "output 1 when exactly one of two binary inputs is on." The book was read, not entirely fairly, as a death sentence for the whole approach, and funding for neural-network research collapsed through most of the 1970s, a period now called the first AI winter. The fix — stacking perceptrons into layers — was known in outline before the winter started; what was missing was a practical way to train a multi-layer network, which had to wait until backpropagation was popularized in 1986. We open with this history because it isn't decoration: everything a single perceptron computes is a straight cut through feature space, and everything it cannot compute follows directly from that one fact.

Inputs, weights, bias, and the dot product

Say a clinic runs a breast-cancer screening panel and measures, from a digitized image of a cell nucleus, two numbers per sample: mean_radius and mean_texture. Each patient sample is a vector x = [x1, x2]. A perceptron holds one weight per input feature, w = [w1, w2], plus a single scalar bias, b. It computes a weighted sum of the inputs — the dot product — and adds the bias:

z = w·x + b = w1·x1 + w2·x2 + b

The weights say how much each feature should count toward the verdict, and the sign matters: a positive w1 means larger mean_radius pushes toward one class, a negative weight pushes toward the other. The bias shifts the whole decision independent of the inputs — without it, every decision boundary would be forced to pass through the origin, which is rarely where real data sits. In our screening example, malignant nuclei tend to run larger and more irregular, so a trained perceptron typically ends up with positive weights on both radius and texture.

The step rule and the geometry of the boundary

The raw sum z is an unbounded real number. The perceptron turns it into a decision with the step function: predict class 1 if z ≥ 0, else predict class 0. That threshold at zero is not arbitrary — it is exactly the condition w·x + b = 0, which is the equation of a line in two dimensions (a hyperplane in general, for however many features you have). Every point on one side of that line gets pushed positive and classified malignant; every point on the other side gets pushed negative and classified benign. Changing w rotates the line; changing b slides it without rotating it. This is the single most important geometric fact in this unit: a perceptron is a linear classifier, full stop. It can only ever separate two classes with a straight line, plane, or hyperplane — never a curve. If your two tumor classes happen to be separable by a straight cut through radius-and-texture space, a perceptron will find it. If they are not — if malignant and benign samples interleave in a way no straight line can cleanly divide — no amount of retraining will fix that; the model class itself is too weak.

The perceptron learning rule

Training adjusts w and b using only the samples the model currently gets wrong. For each training example (x, y) with true label y ∈ {0, 1}, compute the current prediction ŷ, then update:

w ← w + η·(y − ŷ)·x b ← b + η·(y − ŷ)

Here η (eta) is the learning rate, a small positive constant controlling step size. Notice what happens when the prediction is already correct: y − ŷ = 0, and nothing changes. When the model wrongly predicts benign for a malignant sample, y − ŷ = 1, and the update nudges w in the direction of x, tilting the boundary toward correctly capturing that point next time. Run this over the training set repeatedly (an epoch is one full pass) and, for data that is truly linearly separable, the perceptron convergence theorem guarantees this process finds a separating line in a finite number of updates. That guarantee is also the rule's whole weakness: it says nothing about what happens when the data is not linearly separable, which is most real clinical data with more than a couple of features.

Exactly what XOR breaks

Minsky and Papert's XOR argument generalizes directly to screening data. Imagine a (contrived but illustrative) rule: flag a sample as high-risk only when a lab value is either high glucose or high blood pressure, but not both — perhaps because "both elevated" gets routed to a different, already-flagged pathway. Plot the four corners of that truth table and you'll find no single straight line separates the two flagged corners from the two unflagged ones; the positive class sits on two diagonally opposite corners. Any straight cut you draw either merges both positives with a negative or leaves one positive stranded on the wrong side. This isn't a training failure to be fixed with more epochs or a smaller learning rate — it is a hard limit on what one linear boundary can express. The way out, previewed here and built in the next two lessons, is to stack more than one such unit and pass the result through a nonlinearity, which is exactly what turns a collection of lines into a boundary that can bend.

import numpy as np

# Two-feature screening split: mean_radius, mean_texture (standardized)
# y = 1 -> malignant, y = 0 -> benign
X = np.array([
    [1.8, 1.2], [2.1, 0.9], [1.5, 1.6], [2.4, 1.1],   # malignant
    [-1.2, -0.8], [-0.9, -1.4], [-1.6, -0.5], [-0.7, -1.1],  # benign
])
y = np.array([1, 1, 1, 1, 0, 0, 0, 0])

w = np.zeros(2)
b = 0.0
eta = 0.1

def step(z):
    return 1 if z >= 0 else 0

for epoch in range(20):
    mistakes = 0
    for xi, yi in zip(X, y):
        z = np.dot(w, xi) + b
        y_hat = step(z)
        error = yi - y_hat
        if error != 0:
            w += eta * error * xi
            b += eta * error
            mistakes += 1
    if mistakes == 0:
        print(f"converged after {epoch} epochs, w={w}, b={b:.2f}")
        break

Only misclassified samples move the weights (error != 0); once every sample is on the correct side of the line, mistakes hits zero and the loop stops — the convergence theorem in eight lines.

Key takeaways

  • A perceptron computes w·x + b and thresholds it at zero — the weights encode feature importance and direction, the bias shifts the boundary off the origin.
  • The decision boundary is always a hyperplane — a straight line in 2D, a flat plane in 3D, and so on — regardless of how the weights are tuned.
  • The learning rule only updates on mistakes: w ← w + η(y − ŷ)x, which is why training halts automatically once every point is correctly classified.
  • For linearly separable data, convergence is guaranteed; for anything else — like an XOR-shaped "elevated on exactly one lab value" rule — no amount of training fixes it, because the model class itself cannot bend.
  • The historical lesson matters as much as the math: an entire research program stalled for a decade over a limitation that stacking units and adding nonlinearity resolves.

Lesson 2

Activation functions and why nonlinearity is the whole point

An activation function is a fixed, elementwise nonlinear function applied to the output of a linear layer before that output is passed to the next layer. That definition sounds almost too small to matter — one extra function call sandwiched between two matrix multiplications — but it is the single design choice that separates a neural network from a slightly disguised linear regression. Everything else in this lesson is an explanation of why that one clause, "nonlinear," is load-bearing.

The collapse proof: why stacked linear layers are still linear

Take two layers with no activation between them. The first computes h = W1·x + b1. The second feeds that straight into another affine map: o = W2·h + b2. Substitute:

o = W2·(W1·x + b1) + b2 = (W2·W1)·x + (W2·b1 + b2)

Define W' = W2·W1 and b' = W2·b1 + b2, and you get o = W'·x + b' — a single affine map, indistinguishable in form from one perceptron layer. This holds no matter how many linear layers you stack: ten linear layers collapse, by the same substitution repeated, into one linear layer with different weights. A ten-layer network with no activations has exactly the representational power of Lesson 1's single perceptron — it can only ever cut feature space with one hyperplane, no matter how many "layers" of parameters you throw at diabetes risk prediction from fasting glucose, HbA1c, BMI, and blood pressure. Depth without nonlinearity buys you nothing but wasted computation. This is the proof that motivates everything else in deep learning: an activation function has to be inserted after every linear layer (except usually the last) or the network never escapes being a linear model in an expensive disguise.

The standard hidden activations

The sigmoid function, σ(z) = 1 / (1 + e^-z), squashes any real input into (0, 1) and was the default choice through the 1980s and 1990s because its output reads like a probability and its derivative has a clean closed form, σ(z)·(1 − σ(z)). Its problem is saturation: for |z| > ~4, the curve is nearly flat, so the derivative shrinks toward zero. In a network with several hidden layers, chaining several near-zero derivatives together during backpropagation makes earlier layers receive a vanishingly small learning signal — the vanishing gradient problem. Tanh, (e^z − e^-z)/(e^z + e^-z), is sigmoid's zero-centered cousin, squashing to (-1, 1); zero-centered outputs help optimization slightly but tanh saturates in exactly the same way at its extremes.

ReLU (rectified linear unit), f(z) = max(0, z), became the default for hidden layers in the early 2010s because it doesn't saturate for positive inputs — its derivative is a constant 1 wherever z > 0 — and it's essentially free to compute, one comparison per unit. Its failure mode is the mirror image: for any input where z < 0, both the output and the gradient are exactly zero. If a unit's weights drift so that it outputs a negative pre-activation for every sample in your training set — plausible on an imbalanced diabetes cohort where one lab-value combination rarely occurs — that unit's gradient is permanently zero and it stops learning entirely, a dead ReLU. Leaky ReLU patches this with a small nonzero slope for negative inputs, typically f(z) = z if z > 0 else 0.01·z, so a unit that drifts negative still receives a small gradient and has a chance to recover. GELU (Gaussian Error Linear Unit), used in transformer architectures like BERT and GPT, is a smooth curve that weights inputs by their value under the standard normal CDF — roughly ReLU with the sharp corner at zero rounded off — and tends to train slightly better in very deep networks, at the cost of being more expensive to compute exactly.

Output-layer activation is a different decision

Hidden-layer activations exist to add nonlinearity between layers; output-layer activations exist to shape the network's final number into the form the task needs, and the right choice depends entirely on what you're predicting. For a single yes/no clinical call — malignant or benign — the output layer is one unit with a sigmoid, producing a single probability in (0, 1). For a multi-class call — say, classifying a lab panel into "normal," "prediabetic," or "diabetic" — the output layer needs softmax across three units, which exponentiates each unit's score and normalizes so the three outputs sum to exactly 1, giving a proper probability distribution over mutually exclusive classes (sigmoid applied independently to three units would not sum to 1 and would treat the classes as non-exclusive). For a regression target — predicting an actual continuous HbA1c value rather than a risk category — the output layer uses no activation at all (a "linear" output), because squashing to (0, 1) would make it impossible for the network to ever predict a value outside that range. Mixing these up is a common and costly bug: a sigmoid output on a three-class problem, or a softmax on a regression target, will train "successfully" in the sense that loss goes down, while producing outputs that are structurally wrong for the task.

import numpy as np

def sigmoid(z):    return 1 / (1 + np.exp(-z))
def sigmoid_d(z):  s = sigmoid(z); return s * (1 - s)

def tanh(z):        return np.tanh(z)
def tanh_d(z):       return 1 - np.tanh(z) ** 2

def relu(z):         return np.maximum(0, z)
def relu_d(z):       return (z > 0).astype(float)

def leaky_relu(z, a=0.01):   return np.where(z > 0, z, a * z)
def leaky_relu_d(z, a=0.01): return np.where(z > 0, 1.0, a)

# Sweep a range wide enough to show saturation on both tails
z = np.linspace(-6, 6, 13)

for name, f, d in [("sigmoid", sigmoid, sigmoid_d),
                    ("tanh", tanh, tanh_d),
                    ("relu", relu, relu_d),
                    ("leaky_relu", leaky_relu, leaky_relu_d)]:
    grads = d(z)
    vanish_at = z[np.abs(grads) < 0.01]
    print(f"{name:11s} min grad={grads.min():.4f}  "
          f"near-zero at z={vanish_at}")

Sigmoid and tanh print near-zero gradients at both z ≈ -6 and z ≈ 6; ReLU prints near-zero only for every negative z, exactly the "dead" half; leaky ReLU never drops below a = 0.01 anywhere on the sweep.

Key takeaways

  • Stacked linear layers collapse to one linear layer — W2(W1x+b1)+b2 always simplifies to a single W'x+b', so nonlinearity is what makes depth meaningful, not optional decoration.
  • Sigmoid and tanh saturate on both tails, driving gradients toward zero and slowing or stalling learning in deeper stacks; ReLU saturates only on the negative side, and can go fully dead there.
  • Leaky ReLU and GELU exist specifically to patch ReLU's dead-unit failure mode, at the cost of a hyperparameter or extra compute respectively.
  • Output-layer activation depends on the prediction task, not on convention: sigmoid for one binary probability, softmax for mutually exclusive multi-class probabilities, no activation for unbounded regression.
  • A model can appear to train fine with the wrong output activation — loss decreases — while producing predictions that are structurally invalid for the task.

Lesson 3

Layers, width, and depth: stacking into a network

Think of a single layer the way you'd think of one stage in a lab's sample-processing pipeline. A phlebotomist draws blood and hands over a vial of raw material — the input features. The first workstation doesn't diagnose anything; it just recombines the raw numbers into a new panel of derived readings, the way a chemistry analyzer turns raw absorbance values into glucose and cholesterol figures. Those derived readings go to the next workstation, which recombines them again into something more specific — maybe an index that blends three lab values into one metabolic-risk score. Only at the very last station does someone actually stamp a verdict: high risk or not. Each workstation only ever does two things — recombine what it received, then apply a fixed rule to decide what to pass forward — and it's that repeated recombine-then-decide pattern, chained station after station, that a neural network's layers are named after.

A layer as an affine map plus an activation

Formally, one layer takes an input vector, applies a linear (affine) transformation, and then applies an activation function elementwise:

h = f(W·x + b)

Here x is the input vector, W is a weight matrix, b is a bias vector, and f is the activation from Lesson 2 (ReLU, sigmoid, and so on). The output h is called a hidden layer when it feeds into further layers rather than being the network's final answer. Stack several of these — h1 = f(W1·x + b1), then h2 = f(W2·h1 + b2), then a final output layer — and you have a multi-layer perceptron (MLP), the plain "fully connected" network that everything more elaborate in this course builds on.

Shape bookkeeping: batch and features

In practice you never push one patient through the network at a time; you push a batch of samples together as a matrix, because matrix multiplication over a batch is dramatically faster than a Python loop over individual rows. Suppose you're screening on a diabetes-risk panel with 8 lab features (fasting glucose, HbA1c, BMI, systolic BP, diastolic BP, triglycerides, HDL, age) and you process 32 patients at once. Your input is a matrix of shape (32, 8) — batch dimension first, feature dimension second, by near-universal convention. A layer that maps 8 features down to 16 hidden units has a weight matrix W of shape (8, 16) and a bias vector of shape (16,). The matrix multiply x @ W takes (32, 8) @ (8, 16) → (32, 16) — the batch dimension rides along unchanged while the feature dimension transforms. Get a single dimension wrong in this bookkeeping — transpose W, or confuse which axis is batch — and the failure is usually a shape-mismatch crash, which is in fact the friendly outcome; the unfriendly outcome is a shape that happens to be compatible via broadcasting but silently computes the wrong thing.

Counting parameters

Every weight matrix and bias vector is a set of learned numbers, and counting them tells you how much capacity — and how much data — a network needs. A layer mapping n_in features to n_out units has n_in × n_out weights plus n_out biases:

params = n_in·n_out + n_out

For the 8-feature-to-16-unit layer above: 8×16 + 16 = 144 parameters. A second hidden layer from 16 units to 8 units adds 16×8 + 8 = 136. A final output layer from 8 units to 1 (a single malignancy-probability logit) adds 8×1 + 1 = 9. Total: 289 parameters for a small three-layer network — trivial by modern standards, but the arithmetic scales linearly, and it's worth internalizing early because it's the first thing to check when a network is overfitting a few hundred labeled patient records: a network with more parameters than training examples is memorizing, not learning.

What hidden units learn, and width versus depth

No one hand-assigns meaning to a hidden unit; each one's weight vector is just whatever the training process settled on. But empirically, in a network trained on tabular clinical features, individual hidden units often end up sensitive to combinations that a clinician would recognize — one unit might respond strongly to "high glucose and high BMI together," effectively synthesizing a metabolic-syndrome-like feature the raw inputs never explicitly encoded. Width — more units per layer — gives a layer more of these learned feature-combinations to work with in parallel. Depth — more layers — lets the network build combinations of combinations: a first layer might learn "glucose-and-BMI" and "BP-and-age" as two separate hidden features, and a second layer can then learn to combine those two derived features into a still higher-level risk indicator. Wide-shallow networks tend to be easier to train and reason about but often need disproportionately more units to represent something that depth would build in two cheap stages. Deep-narrow networks are more parameter-efficient for compositional patterns but harder to train (vanishing gradients from Lesson 2 get worse the deeper you go) and easier to misconfigure.

The theoretical backstop for all of this is the universal approximation theorem: a feedforward network with a single hidden layer of sufficient width can approximate any continuous function on a bounded input domain to arbitrary precision. It is tempting to read that as "one hidden layer is all you ever need," and that reading is wrong in a way that matters. The theorem proves existence — that some set of weights achieves the approximation — it says nothing about whether ordinary gradient-based training can actually find those weights, and "sufficient width" can mean a number of units that is computationally absurd for anything but toy problems. Depth is not something the theorem rules out as unnecessary; in practice, depth is very often how you get a network that both fits well and is learnable with the training procedures Unit 2 and Unit 3 will cover. Existence is not the same thing as learnability, and confusing the two is one of the more common overclaims in introductions to this material.

import numpy as np

# Forward pass through a 2-hidden-layer MLP for diabetes-risk screening.
# Features: glucose, HbA1c, BMI, systolic_bp, diastolic_bp, triglycerides, HDL, age
batch_size, n_features = 32, 8
h1_units, h2_units, n_out = 16, 8, 1

rng = np.random.default_rng(0)
X = rng.normal(size=(batch_size, n_features))          # (32, 8)

W1 = rng.normal(scale=0.1, size=(n_features, h1_units)) # (8, 16)
b1 = np.zeros(h1_units)                                  # (16,)
W2 = rng.normal(scale=0.1, size=(h1_units, h2_units))    # (16, 8)
b2 = np.zeros(h2_units)                                  # (8,)
W3 = rng.normal(scale=0.1, size=(h2_units, n_out))       # (8, 1)
b3 = np.zeros(n_out)                                     # (1,)

def relu(z):
    return np.maximum(0, z)

def sigmoid(z):
    return 1 / (1 + np.exp(-z))

h1 = relu(X @ W1 + b1)      # (32, 8) @ (8, 16)  -> (32, 16)
h2 = relu(h1 @ W2 + b2)     # (32, 16) @ (16, 8) -> (32, 8)
out = sigmoid(h2 @ W3 + b3) # (32, 8) @ (8, 1)   -> (32, 1)  risk probability

for name, W, b in [("layer1", W1, b1), ("layer2", W2, b2), ("layer3", W3, b3)]:
    n = W.size + b.size
    print(f"{name}: W{W.shape} + b{b.shape} = {n} parameters")
print("output shape:", out.shape)

Each print line shows a shape transition ((8,16) then (16,8) then (8,1)) and its parameter count; the final out.shape confirms the batch dimension, 32, survived unchanged while the feature dimension collapsed to a single risk score per patient.

Key takeaways

  • A layer is f(W·x + b) — an affine transform followed by a fixed activation — and stacking these is the entire definition of a multi-layer perceptron.
  • Shapes carry a batch dimension and a feature dimension; a layer's weight matrix shape is (n_in, n_out), and getting an axis wrong produces either a crash or, worse, a silent broadcast that computes garbage.
  • Parameter count per layer is n_in·n_out + n_out — arithmetic worth doing before training, since more parameters than labeled patient records is a memorization risk, not a capacity win.
  • Width adds parallel learned feature-combinations; depth composes combinations of combinations — neither is strictly better, and depth trades easier training for parameter efficiency.
  • Universal approximation proves existence of a solution, not that training will find it — a single wide-enough hidden layer can approximate any continuous function in theory, but "sufficient width" can be computationally impractical, and gradient descent is not guaranteed to locate the right weights.

Unit 1 quiz

Seven questions covering the perceptron's geometry and learning rule, activation behavior and saturation, and layer shapes and parameter counting.

Question 1 Multiple choice

A perceptron is trained on mean_radius and mean_texture to separate malignant from benign nuclei. What shape can its decision boundary take?

  1. Any smooth curve that best fits the data
  2. A straight line only
  3. A step function with multiple thresholds
  4. A circle centered on the class mean
Show answer

Answer: B. The decision rule is w·x + b ≥ 0, and the boundary where that expression equals zero is, in two dimensions, exactly a straight line. In higher dimensions it generalizes to a flat hyperplane — never a curve.

A is wrong because "best fits the data" implies the model can bend to match the data's shape; a perceptron's boundary is fixed to be linear regardless of how well or poorly that fits the true class boundary. Training changes the line's position and angle, never its straightness.

Question 2 True / false

The perceptron learning rule updates the weight vector on every training example, whether or not the current prediction is correct.

  1. True
  2. False
Show answer

Answer: False. The update is w ← w + η(y − ŷ)x. When the prediction is already correct, y − ŷ = 0, and the update term vanishes — the weights are left unchanged. Updates only happen on misclassified examples.

It's tempting to think every layer of "training" must touch every example equally, since that's how some other model-fitting procedures work, but the perceptron rule is specifically mistake-driven: it is exactly this property that lets it halt automatically (zero mistakes in a full pass) once a separating line is found.

Question 3 Fill in the blank

Minsky and Papert's 1969 book showed that a single perceptron cannot learn the   function, because its positive cases sit on diagonally opposite corners of the input space with no single straight line able to separate them from the negative cases.

Show answer

Answer: XOR (exclusive or). The classic example is two binary inputs where the output is 1 only when exactly one input is on — the two "1" outputs and two "0" outputs form an X-shaped pattern that no single hyperplane can cut cleanly in two.

A common wrong guess is "AND" or "OR" — both of those are linearly separable (their positive cases sit together on one side), which is exactly why they were not the example Minsky and Papert used to make their point.

Question 4 Multiple choice

Why does stacking three linear layers with no activation function between them fail to add any representational power over a single linear layer?

  1. Because linear layers cannot be trained with gradient descent
  2. Because the composition of affine maps is itself always a single affine map
  3. Because three layers always overfit a small clinical dataset
  4. Because linear layers require an odd number of hidden units
Show answer

Answer: B. W2(W1x + b1) + b2 algebraically simplifies to (W2W1)x + (W2b1 + b2), which has the same form as a single layer's W'x + b'. This holds for any number of stacked linear layers — the collapse is exact, not approximate.

A is wrong: linear layers train fine with gradient descent on their own (that's ordinary linear regression via gradient descent) — the issue here is representational power, not trainability. Overfitting (C) is a separate concern about parameter count versus data size, unrelated to whether activations are present.

Question 5 Code completion

Complete the ReLU derivative function so that it correctly reports a zero gradient exactly where ReLU is flat.

def relu_d(z):
    return ______________
  1. np.ones_like(z)
  2. (z > 0).astype(float)
  3. np.maximum(0, z)
  4. 1 / (1 + np.exp(-z))
Show answer

Answer: B. ReLU's derivative is 1 wherever the pre-activation z is positive and 0 wherever it's negative (undefined exactly at zero, conventionally taken as 0 or 1). (z > 0).astype(float) produces exactly that array of 1s and 0s.

C, np.maximum(0, z), is ReLU itself, not its derivative — a common mix-up when writing both functions quickly. D is the sigmoid function, unrelated to ReLU's derivative and never zero for any finite z, which would incorrectly suggest ReLU never saturates.

Question 6 Multiple choice

You're building the output layer for a model that classifies a lab panel into exactly one of three mutually exclusive categories: normal, prediabetic, diabetic. Which output-layer activation is appropriate?

  1. A single sigmoid unit
  2. Softmax across three units
  3. No activation (linear output)
  4. ReLU across three units
Show answer

Answer: B. Softmax exponentiates each of the three units' scores and normalizes them to sum to 1, producing a valid probability distribution over mutually exclusive classes — exactly the structure "exactly one of three categories" requires.

A single sigmoid (A) only produces one probability and is the right choice for a binary yes/no call, not a three-way exclusive choice. ReLU (D) is never used on an output layer meant to represent probabilities, since its output is unbounded above and exactly zero for half its input range — neither behavior matches "probability of a class."

Question 7 Bug fix

A diabetes-risk network takes a batch of 32 patients with 8 lab features each. The first hidden layer is supposed to produce 16 hidden units per patient. Find the bug in the shape setup.

X = np.random.randn(32, 8)          # (batch=32, features=8)
W1 = np.random.randn(16, 8) * 0.1   # intended: maps 8 features -> 16 units
b1 = np.zeros(16)

h1 = relu(X @ W1 + b1)              # crashes: shapes (32,8) and (16,8) not aligned
  1. b1 should be shape (32,) instead of (16,)
  2. W1 should be shape (8, 16) instead of (16, 8)
  3. X should be transposed to shape (8, 32)
  4. ReLU cannot be applied to a batch of inputs
Show answer

Answer: B. Matrix multiplication X @ W1 requires X's last dimension to match W1's first dimension: (32, 8) @ (8, 16) → (32, 16). With W1 as (16, 8), the inner dimensions (8 vs 16) don't line up and NumPy raises a shape-mismatch error. The weight matrix for an n_in → n_out layer must be shaped (n_in, n_out).

C would technically make a different multiplication valid but destroys the batch convention (batch dimension must ride first through every layer) and would break every other layer downstream that expects a (batch, features) input. A is wrong because the bias only needs one value per output unit (16), broadcast across the batch — it should never scale with batch size.
