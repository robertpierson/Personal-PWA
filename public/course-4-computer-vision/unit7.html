Lesson 1

Generative modelling and GANs

A discriminative model answers a narrower question than people give it credit for. Every classifier in Units 1–6 — the defect detector, the segmentation U-Net, the object detector — learns p(y | x): given this image, what's the label, mask, or box? It never has to know what a valid image looks like, only how to carve up the ones it's handed. A generative model is answering the harder, symmetric question: it estimates p(x), the distribution the images themselves were drawn from, well enough to draw new samples from it. That's a qualitatively bigger ask. A discriminative model can cheat by learning a decision boundary that happens to work; a generative model has to internalize enough of the structure of "photograph of a product on a shelf" or "microscope slide with a tumor margin" that it can hallucinate a convincing new instance from scratch. If it gets the joint statistics wrong — plausible textures assembled into an impossible object — the fakery shows.

Three families have taken serious runs at p(x), and it's worth a fast map before diving into the one this lesson spends its time on. Variational autoencoders (VAEs) extend the encoder/decoder you met in Unit 5: an encoder maps an image to a distribution over a latent code (typically a mean and variance for each latent dimension, not a single point), you sample from that distribution, and a decoder reconstructs an image from the sample. Training combines a reconstruction loss (does the decoded image match the input?) with a KL-divergence term that pulls the encoded distribution toward a standard normal, which is what makes the latent space usable for sampling — you can draw a fresh z ~ N(0, I) at generation time and decode it without ever having seen that exact code during training. The catch is well known: VAE samples are characteristically blurry. Minimizing reconstruction error with a pixel-wise loss (typically mean squared error) rewards the average of plausible outputs over any single sharp one, so fine texture — the grain in a fabric, the crisp edge of a label print — gets smoothed away. Autoregressive models like PixelCNN take the opposite approach: generate the image one pixel at a time, each conditioned on every pixel already generated, using a masked convolution so the model can never see "future" pixels during training. This produces sharp, high-fidelity samples but at a brutal cost — generating a 256×256 image means 65,536 sequential network evaluations, which is why autoregressive pixel models never scaled to production image generation the way they did to text.

The GAN game

Generative adversarial networks (Goodfellow et al., 2014) sidestep both problems with a trick that doesn't estimate p(x) explicitly at all. Two networks are trained against each other: a generator G maps random noise z to an image G(z), and a discriminator D looks at an image and outputs the probability it's real rather than generator-produced. D is trained to get better at telling real product photos from generated ones; G is trained to fool D. Neither network ever computes a likelihood — the training signal is entirely "did the other network get fooled." This is the minimax game:

min_G max_D E[log D(x)] + E[log(1 − D(G(z)))]

At the (theoretical) equilibrium, G produces samples indistinguishable from real data and D outputs 0.5 everywhere — it can no longer do better than a coin flip. In practice this equilibrium is notoriously hard to reach, and the failure modes are specific enough to name. Mode collapse is the generator finding one or a handful of outputs that reliably fool the current discriminator and producing only those — for a synthetic-defect generator this looks like every generated "scratch" defect being a near-identical diagonal line, useless for augmenting a classifier that needs variety. Non-convergence is the two networks oscillating rather than settling — G learns to beat last round's D, D adapts, G shifts again, forever, with no fixed point. Vanishing gradients happen when D becomes too good too early: if D confidently rejects every generated sample, log(1 − D(G(z))) saturates and its gradient with respect to G's parameters shrinks toward zero, so G stops receiving useful signal to improve — the discriminator's competence starves its own training partner.

Stabilizing the game

The fixes that stuck target the loss geometry directly rather than tweaking learning rates. Wasserstein GAN (WGAN) replaces the original JS-divergence-flavored objective with the Earth Mover's (Wasserstein) distance between real and generated distributions, which stays smooth and informative even when the two distributions barely overlap — exactly the regime where the original loss vanishes. Enforcing the Lipschitz constraint the Wasserstein formulation requires was originally done by clipping discriminator weights, a coarse hack; WGAN-GP (gradient penalty) instead penalizes the discriminator whenever the gradient norm of its output with respect to interpolated real/fake inputs strays from 1, which is both more stable and doesn't cripple the discriminator's capacity the way weight clipping does. Spectral normalization is a complementary fix applied directly to the discriminator's weight matrices: dividing each weight matrix by its largest singular value at every forward pass caps the Lipschitz constant of each layer without any extra loss term, and it composes cleanly with almost any GAN variant. On top of a stabilized base, conditional GANs feed a class label or embedding into both G and D so generation can be steered ("generate a scratch defect on brushed aluminum" rather than "generate something"), and StyleGAN's contribution was architectural: injecting the latent code at every resolution level of the generator via adaptive instance normalization, which disentangles coarse structure (pose, layout) from fine detail (texture, color) and gives a designer separate knobs for each.

def gan_training_step(G, D, real_images, opt_G, opt_D, z_dim=128):
    batch_size = real_images.shape[0]
    z = torch.randn(batch_size, z_dim)

    # --- 1. Discriminator step ---
    # D learns to separate real product photos from G's fakes.
    fake_images = G(z).detach()          # detach: no generator grad here
    d_real = D(real_images)
    d_fake = D(fake_images)
    loss_D = -(torch.log(d_real + 1e-8).mean()
               + torch.log(1 - d_fake + 1e-8).mean())
    opt_D.zero_grad(); loss_D.backward(); opt_D.step()

    # --- 2. Generator step ---
    # G learns to make D(G(z)) large -- i.e. fool the discriminator.
    z = torch.randn(batch_size, z_dim)
    fake_images = G(z)
    d_fake = D(fake_images)
    loss_G = -torch.log(d_fake + 1e-8).mean()   # non-saturating variant
    opt_G.zero_grad(); loss_G.backward(); opt_G.step()

    return loss_D.item(), loss_G.item()

Two separate backward passes per step, one per network. Note fake_images.detach() in the discriminator step — it stops gradient from flowing into G during D's update — and the non-saturating -log(D(G(z))) generator loss, which avoids the vanishing-gradient failure of the original log(1 - D(G(z))) formulation when D is already confident.

Key takeaways

  • Generative vs. discriminative is a distribution question: discriminative models learn p(y|x), generative models learn (or learn to sample from) p(x) — a strictly harder target, which is why generative failures look like "plausible pieces, impossible whole" rather than a wrong label.
  • VAEs trade sharpness for a well-behaved latent space via the reconstruction+KL objective; their blur is a direct consequence of pixel-wise reconstruction loss, not a fixable bug.
  • GAN failure modes have names because they have causes: mode collapse (generator exploits a narrow trick), non-convergence (no fixed point), vanishing gradients (an over-strong discriminator starves its own signal) — each has a matched fix (minibatch discrimination / diversity losses, WGAN-GP, non-saturating loss).
  • WGAN-GP and spectral normalization stabilize training by controlling the discriminator's Lipschitz constant, not by changing the generator architecture at all.
  • StyleGAN's disentangled per-resolution latent injection is what makes "same layout, different texture" controllable generation possible — a capability plain GANs don't offer.

Lesson 2

Diffusion models

Picture a sculptor who starts not with marble but with a block of pure static — visual noise with no structure at all — and works backward, one small correction at a time, coaxing a recognizable form out of the mess. Nudge the noise slightly toward "less noisy," repeat a few hundred times, and a coherent image emerges at the end. That is, almost literally, how a diffusion model generates an image, and it is the mechanism behind essentially every state-of-the-art image generator as of this writing — Stable Diffusion, Midjourney's and DALL·E's underlying samplers, and the synthetic-data engines increasingly used to manufacture rare-class training examples all sit on this same backbone.

The trick starts in reverse from how you'd generate: define a fixed, un-learned forward noising process that takes a real image x₀ and, over T steps (often 1,000), adds a small amount of Gaussian noise at each step until x_T is indistinguishable from pure noise. Because each step just adds Gaussian noise, there's a closed-form shortcut to jump straight from x₀ to any noise level t without simulating every intermediate step: x_t = √(ᾱ_t)·x₀ + √(1−ᾱ_t)·ε, where ε ~ N(0,I) is the noise actually added and ᾱ_t is a scheduled constant that decreases from near 1 (barely noised) to near 0 (almost pure noise) as t grows. The generative half is the reverse process: a neural network learns to undo one small noising step at a time, starting from pure noise at t=T and working down to t=0.

Why predict the noise

You could ask the network to predict the clean image x₀ directly at every step, but in practice it works much better to have it predict the noise ε that was added — this is epsilon-prediction, and it's the standard parameterization from the original DDPM paper (Ho et al., 2020). Once you know ε, recovering an estimate of x₀ or taking one reverse step is simple algebra from the forward equation above; predicting the noise also keeps the target roughly zero-mean and unit-variance at every timestep, which is a much friendlier regression target than a raw image whose statistics shift as t changes. The training objective falls out to something almost embarrassingly simple: sample a real image, sample a random timestep, add the corresponding amount of noise, and train the network to predict that noise with plain mean squared error.

L = E_{t, x₀, ε} [ ‖ ε − ε_θ(x_t, t) ‖² ]

The network doing the predicting, ε_θ, is almost always the U-Net from Unit 5 — the same encoder-decoder-with-skip-connections shape, because denoising needs exactly what segmentation needed: coarse global context (what object, what layout) fused back with fine spatial detail (where exactly the edges are) via the skip connections. The one addition is timestep conditioning: the current step t is embedded (typically a sinusoidal position embedding, echoing Course 2's positional encodings) and injected into every U-Net block, usually by modulating the normalization statistics, so a single network can denoise a nearly-clean image at t=5 and a nearly-pure-noise image at t=950 with the same weights, just conditioned differently.

Sampling: slow but composable

Generation means running the reverse process: start from x_T ~ N(0,I) and repeatedly predict the noise, subtract a scheduled fraction of it, and (for most of the trajectory) add a controlled amount of fresh noise back in — that last part matters, because without it the sampler behaves more deterministically than the model's training distribution assumes and samples degrade. Doing this the original way, DDPM sampling, uses all T steps and is exactly why diffusion's biggest practical weakness relative to GANs is speed: a GAN generates an image in a single forward pass through G; a naive DDPM sampler needs on the order of 1,000 sequential network evaluations for one image. DDIM (denoising diffusion implicit models) reformulates the reverse process as non-Markovian, which permits skipping steps — sampling with 20-50 steps instead of 1,000 while staying close to DDPM quality — and it's the default in virtually every production system today. Even at 20-50 steps, that's still 20-50x more network evaluations than a GAN's one, which is the honest trade-off: diffusion models are typically more stable to train and produce higher-fidelity, more diverse samples, but they are markedly slower at inference, which matters directly for a design tool where a user is waiting on a preview.

Classifier-free guidance is the mechanism that lets you dial how strongly a diffusion model obeys a conditioning signal (a text prompt, a class label) versus how strongly it just produces "any plausible image." During training, the conditioning is randomly dropped some fraction of the time, so the same network learns both a conditional ε_θ(x_t, t, c) and an unconditional ε_θ(x_t, t). At sampling time you compute both and extrapolate away from the unconditional prediction toward the conditional one: ε̂ = ε_θ(x_t,t) + w·(ε_θ(x_t,t,c) − ε_θ(x_t,t)), where guidance weight w > 1 pushes samples to match the condition more aggressively, at some cost to diversity and, past a point, to realism (oversaturated colors, artifacts). Finally, running full diffusion in pixel space is expensive simply because images are big — a 512×512×3 tensor denoised 50 times is a lot of compute. Latent diffusion (the technique behind Stable Diffusion) sidesteps this by first compressing the image with a VAE-style encoder (much like Lesson 1's VAE, though usually without the KL term dominating) into a much smaller latent grid — say 64×64×4 instead of 512×512×3 — running the entire noising/denoising process in that compressed latent space, and only decoding back to pixels once, at the very end. This is what made high-resolution text-to-image generation tractable on consumer hardware.

def diffusion_training_step(unet, x0, opt, T=1000, alpha_bar=None):
    batch_size = x0.shape[0]

    # 1. sample a random timestep per image in the batch
    t = torch.randint(0, T, (batch_size,))

    # 2. sample noise and build the noised image x_t
    eps = torch.randn_like(x0)
    a_bar_t = alpha_bar[t].view(-1, 1, 1, 1)      # per-sample schedule value
    x_t = torch.sqrt(a_bar_t) * x0 + torch.sqrt(1 - a_bar_t) * eps

    # 3. U-Net predicts the noise, conditioned on t
    eps_pred = unet(x_t, t)

    # 4. plain MSE between true and predicted noise
    loss = torch.mean((eps - eps_pred) ** 2)

    opt.zero_grad(); loss.backward(); opt.step()
    return loss.item()

One network, one loss, no adversary — contrast this with the two-optimizer dance in Lesson 1's GAN step. The only moving part is which timestep t gets sampled, which is why diffusion training is markedly more stable than GAN training in practice.

Key takeaways

  • The forward process is fixed and un-learned — only the reverse denoising process is a trained network — and the closed-form shortcut x_t = √ᾱ_t·x₀ + √(1−ᾱ_t)·ε lets training sample any timestep directly.
  • Epsilon-prediction plus plain MSE is the whole training objective; it works because predicting zero-mean noise is a better-conditioned regression target than predicting a raw image whose statistics vary with t.
  • The U-Net backbone is unchanged from Unit 5 except for timestep conditioning injected into every block — the same coarse+fine fusion that served segmentation now serves denoising.
  • DDIM's non-Markovian reformulation buys a 20-50x sampling speedup over DDPM by skipping steps, but diffusion is still far slower per image than a GAN's single forward pass — a real deployment cost, not a footnote.
  • Latent diffusion moves the entire denoising loop into a compressed VAE latent space, which is the difference between "runs on a data-center cluster" and "runs on a single consumer GPU."

Lesson 3

Conditioning, control, and the honest limits

Two people want two very different things from the same generator, and neither wants "any plausible image." A designer at a furniture brand needs a photoreal render of this exact armchair, in walnut, on a specific studio backdrop, not a generic armchair the model invented. A machine-learning engineer building a defect classifier for weld inspection needs synthetic images of the rare defect class that actually resemble the real defects a camera will see on the line — not images that merely look plausible to a human but teach the downstream classifier to key off some artifact of the generator instead of the real physical signature. Both problems are conditioning problems, and both have a failure mode where the output "looks right" until you check whether it's right for the actual task.

Steering generation

The core mechanism for text-to-image conditioning is cross-attention, a direct reuse of the attention mechanism from Course 2: at each U-Net layer, the image features act as queries, and the text prompt's token embeddings (from a frozen or co-trained text encoder) act as keys and values, so every spatial location in the denoising U-Net can attend to whichever words in the prompt are relevant to it. This is strictly more flexible than the single class-embedding conditioning from Lesson 1's conditional GANs, because it lets different regions of the image respond to different parts of an open-ended sentence. But a text prompt alone can't pin down "this exact armchair on this exact backdrop" — text is too coarse an interface for pixel-precise control. Three techniques close that gap. Inpainting masks out a region of an existing image and runs the diffusion process only on the masked pixels, conditioned on the surrounding unmasked context, so you can swap a background while leaving the product untouched. img2img starts the reverse process not from pure noise but from a partially-noised version of a real reference image, so the output stays structurally anchored to that reference while still being subject to the text conditioning. ControlNet goes further: it attaches a trainable copy of (part of) the U-Net that takes an auxiliary structural signal — an edge map, a depth map, a pose skeleton — and injects it into the frozen base model's activations, giving pixel-level layout control (this armchair's silhouette, at this exact angle) while text conditioning still handles style and material.

Synthetic data's domain-gap trap

The rare-defect use case looks like a clean win on paper — generate a thousand synthetic images of a defect you've only photographed twelve times, augment the training set, done. The trap is that a generative model trained on those twelve real examples (or, worse, on a generic pretrained model fine-tuned lightly on them) doesn't just learn "what this defect looks like," it also learns whatever quirks are specific to the generator's own architecture and sampling process — slightly-too-smooth noise textures, characteristic color-channel correlations, subtle checkerboard artifacts from certain upsampling schemes. If the downstream defect classifier is trained on a mix heavy with synthetic images, it can learn to detect those generator fingerprints rather than the physical defect signature, which means it performs beautifully on held-out synthetic data and unpredictably on real camera images from the production line — a sim-to-real domain gap that doesn't show up in ordinary validation because the validation set is contaminated with the same synthetic-only artifacts. The mitigation isn't exotic: keep the real-to-synthetic ratio high, validate exclusively on held-out real images never touched by the generative pipeline, and treat any accuracy jump from adding synthetic data with the same suspicion you'd apply to a suspiciously large accuracy jump from any other data change.

Evaluating generative quality at all is harder than it looks, which is worth being direct about. FID (Fréchet Inception Distance) passes real and generated images through a pretrained Inception network, fits a Gaussian to the feature activations of each set (mean and covariance), and reports the Fréchet distance between the two Gaussians — lower is better, meaning the generated feature distribution sits closer to the real one. Inception Score instead looks only at generated images, rewarding samples that an Inception classifier labels confidently (sharp, recognizable) and that are diverse across classes as a batch. Both have well-documented flaws: FID's Gaussian assumption is a crude approximation to feature distributions that aren't actually Gaussian, both scores are sensitive to which pretrained network and which image preprocessing you use (numbers aren't comparable across papers that used different implementations), and neither score can catch the domain-gap problem above — a generator producing consistent, Inception-flattering fingerprints can score well on FID while still teaching a downstream classifier the wrong thing. Precision-recall for generative models partially addresses FID's blind spot by separating two questions FID conflates into one number: precision asks whether generated samples fall inside the real data manifold (are they realistic?), recall asks whether generated samples cover the full real data manifold (is anything missing — is this the mode collapse problem from Lesson 1, just measured explicitly?). None of these metrics substitute for checking whether the actual downstream task — the defect classifier, the design review — improves; they're diagnostics, not a proof of usefulness.

The authenticity problem

None of this technology is neutral, and the same conditioning and control tools that let a designer swap a product's backdrop let anyone swap a real person's face or voice into a fabricated scene — the deepfake problem is a direct, unavoidable extension of everything in this unit, not a separate topic bolted on. Three lines of defense exist, none complete. Watermarking embeds a signal into generated images — sometimes visible, more often an imperceptible perturbation baked into the generation process itself, so it survives common edits — that a matching detector can read back later; it works only for content generated by cooperating systems, since nothing stops a bad actor from stripping a watermark or using a generator that never adds one. Content provenance standards (such as C2PA) instead attach signed, cryptographic metadata recording an image's origin and edit history to the file — closer to a chain of custody than a mark on the pixels — but metadata is trivially lost the moment an image is re-screenshotted or re-exported through a pipeline that doesn't preserve it. Deepfake detection — classifiers trained to spot generator artifacts, essentially the mirror image of the domain-gap problem above — is explicitly an arms race: each detection technique that publishes its method teaches the next generation of generators exactly which artifact to eliminate, and every generational improvement in fidelity (precisely the kind of progress from GANs to diffusion to latent diffusion charted across this unit) shrinks the artifacts detectors rely on. Recall the hard rule from the top of this course: legitimate use here means design tooling and synthetic training data with a human reviewing anything that leaves the building, never automated impersonation, and never a client-facing deliverable that could pass as an unaltered photograph of a real person without disclosure.

def classifier_free_guidance_step(unet, x_t, t, cond_emb, w=7.5):
    # Two forward passes per step: unconditional and conditional.
    eps_uncond = unet(x_t, t, cond=None)
    eps_cond   = unet(x_t, t, cond=cond_emb)

    # Extrapolate away from unconditional, toward conditional.
    # w > 1 sharpens adherence to the prompt at some cost to diversity.
    eps_guided = eps_uncond + w * (eps_cond - eps_uncond)
    return eps_guided   # fed into the normal DDIM/DDPM update rule

Notice this doubles inference cost per step (two U-Net passes, not one) — a second, separate speed tax layered on top of the multi-step sampling cost from Lesson 2, and one more reason a real product weighs guidance strength against latency.

Key takeaways

  • Cross-attention conditions on open-ended text; ControlNet/inpainting/img2img condition on pixel-precise structure — text alone can't specify an exact product silhouette, which is why production pipelines stack these techniques rather than picking one.
  • Synthetic training data can teach a classifier generator fingerprints instead of the real signal — a domain gap that inflates validation accuracy on synthetic-contaminated test sets while degrading real-world performance, catchable only by validating on real-only held-out data.
  • FID and Inception Score are widely reported and widely flawed — a Gaussian approximation, sensitivity to the scoring network, and no ability to detect generator-fingerprint learning — so treat them as one signal, not a verdict.
  • Precision-recall for generative models separates realism from coverage, giving a way to name mode collapse (low recall) as distinct from poor sample quality (low precision) — FID alone conflates the two.
  • Watermarking, provenance metadata, and detection are each partial and each defeatable; deepfake detection specifically is a genuine arms race that improves as generation improves, never a solved problem to check off.

Question 1 Multiple choice

A team training a GAN to generate synthetic weld-defect images notices that after epoch 40, every generated "porosity" defect looks almost pixel-identical — same size, same position, same shape. Discriminator and generator losses both look numerically stable. What is this, and what's the most direct fix?

  1. Vanishing gradients; lower the discriminator's learning rate.
  2. Mode collapse; add a diversity-encouraging term (e.g. minibatch discrimination) or switch to a loss less prone to it, like WGAN-GP.
  3. Non-convergence; the losses oscillating means training hasn't started yet, so just train longer.
  4. Overfitting; add dropout to the generator.
Show answer

Answer: B. A generator collapsing to near-identical outputs while losses stay numerically stable is the textbook signature of mode collapse — the generator found a narrow trick that reliably fools the current discriminator and stopped exploring. WGAN-GP's smoother loss landscape and explicit diversity-encouraging techniques are the standard countermeasures.

Vanishing gradients (A) present as the generator failing to improve at all, with the discriminator near-perfectly confident — not as a diverse-looking loss curve paired with collapsed outputs. Non-convergence (C) is misdiagnosed here: the losses are described as stable, not oscillating, which is the opposite signature, and "train longer" is not a fix for either problem. Dropout (D) targets overfitting to individual training examples, not collapse onto a narrow output mode; it doesn't address the adversarial dynamic causing this.

Question 2 True / false

Because a VAE's decoder is trained with a pixel-wise reconstruction loss, VAE samples are typically blurrier than GAN or diffusion samples at comparable resolution.

  1. True
  2. False
Show answer

Answer: True. Minimizing a pixel-wise loss like MSE rewards predicting the average of multiple plausible outputs when the model is uncertain, which systematically smooths away fine texture and sharp edges — this is a direct, structural consequence of the objective, not an incidental training artifact that better hyperparameters would fix.

Question 3 Fill in the blank

In DDPM-style diffusion training, the network is trained to predict the   that was added to the image at the sampled timestep, using a   loss between the true and predicted values.

Question 4 Multiple choice

A product team wants to swap a diffusion pipeline from DDPM sampling with 1,000 steps to DDIM sampling with 40 steps for a faster interactive preview. What should they expect?

  1. Identical sample quality with a 25x speedup, since DDIM is mathematically equivalent to DDPM at any step count.
  2. A speedup, with quality close to (though not always exactly matching) the full-step DDPM result, because DDIM's non-Markovian reformulation permits skipping steps.
  3. No speedup, because the U-Net still has to process the full image at every one of the 1,000 original noise levels regardless of sampler.
  4. A speedup only if they also switch to a GAN, since diffusion samplers cannot be accelerated below 1,000 steps.
Show answer

Answer: B. DDIM's reformulation of the reverse process as non-Markovian is precisely what allows it to skip intermediate steps while approximating the same trajectory, typically getting most of DDPM's quality in a fraction of the steps — but "approximating" is the operative word; it is not mathematically identical output.

A overstates the guarantee — DDIM is an approximation, not an equivalence, and quality can degrade at very low step counts. C misunderstands what changes: DDIM changes how many reverse steps are actually run, not merely how they're labeled. D is simply false; DDIM is exactly a same-model acceleration technique that does not require switching architectures.

Question 5 Bug fix

A junior engineer wrote this classifier-free guidance step, but samples come out barely responsive to the prompt no matter how high they set w. Find the bug.

def guidance_step(unet, x_t, t, cond_emb, w=7.5):
    eps_cond   = unet(x_t, t, cond=cond_emb)
    eps_uncond = unet(x_t, t, cond=cond_emb)   # reuse the same call
    eps_guided = eps_uncond + w * (eps_cond - eps_uncond)
    return eps_guided
  1. The guidance weight w should be a value less than 1, not greater.
  2. The unconditional pass is called with cond=cond_emb instead of cond=None, so eps_cond and eps_uncond are identical and the guidance term collapses to zero regardless of w.
  3. The two U-Net calls should share one forward pass to save compute.
  4. The extrapolation direction is reversed — it should be eps_cond - w*(eps_cond - eps_uncond).
Show answer

Answer: B. Both lines call the U-Net with the identical conditioning embedding, so eps_cond - eps_uncond is exactly zero every time, no matter what w is — the model is silently only ever seeing eps_uncond (which happens to equal eps_cond here), which matches the reported symptom of prompt-blindness at any guidance strength. The unconditional call needs cond=None to actually train the divergence between the two predictions that guidance extrapolates along.

A is wrong — the standard useful range for w is greater than 1 (values below 1 under-weight the condition, but that's not the bug reported: at no value of w would this code respond to the prompt). C describes a real optimization some implementations make (batching both passes together), but doing two separate calls correctly would still work, just slower — it isn't the cause of the reported bug. D proposes a different, also-valid-looking formula, but it doesn't address the actual defect, which is that the two branches are identical inputs, not a wrong arithmetic sign.

Question 6 Multiple choice

A hospital-imaging team generates synthetic X-rays of a rare fracture pattern with a fine-tuned diffusion model to augment a scarce real dataset. After adding the synthetic images, their held-out test accuracy (drawn from the same synthetic pipeline) jumps from 81% to 97%. What should this jump make them suspicious of?

  1. Nothing — a 16-point jump confirms the augmentation worked and the classifier is ready to deploy.
  2. That the FID score of the generator must have been poor, since FID would have caught this.
  3. That the classifier may have learned to key off generator-specific artifacts shared between the synthetic training and synthetic test images, rather than the real fracture signature — the held-out set is contaminated with the same synthetic fingerprints.
  4. That classifier-free guidance was set too low during generation.
Show answer

Answer: C. A large accuracy jump on a test set that itself contains synthetic images is exactly the domain-gap trap: the classifier can be rewarded for detecting shared generator fingerprints rather than the true physical signature, and this failure is invisible until validated against real images the generator never touched.

A ignores the specific risk named in the lesson — a suspicious jump warrants scrutiny of the evaluation set's composition, not automatic celebration. B is wrong on the mechanics: FID measures how close generated images are to real ones in feature-statistic space, and it specifically cannot detect a classifier learning to exploit shared fingerprints between synthetic train and synthetic test data — that's a blind spot named explicitly in the lesson, not something FID screens for. D is a non sequitur; guidance strength affects prompt adherence and sample diversity, not this particular contamination failure mode.

Question 7 True / false

FID and Inception Score, computed correctly, are sufficient on their own to certify that a set of generated images will actually improve a downstream classifier's real-world performance.

  1. True
  2. False

Answer: False. Answer: B. ControlNet's trainable branch is injected into the frozen base model's activations additively (the original paper zero-initializes the injection so it starts as a no-op and gradually learns useful contributions), so the base network's learned image priors and the auxiliary structural signal both influence the decoder.

A discards the entire point of ControlNet — without adding control_features in, the edge map has no path to affect generation at all.
